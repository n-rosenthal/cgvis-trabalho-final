**  implementando /rochas procedurais/
*** Geração Procedural de Esferas
A função =ProceduralRock::generateSphere()= é responsável por construir a malha triangular utilizada para representar uma rocha aproximadamente esférica. A abordagem gera uma esfera parametrizada em coordenadas esféricas, visando produzir uma geometria compatível com o modelo de colisão esférico adotado pelo sistema físico.

*** Parametrização Esférica
Cada vértice da esfera é obtido a partir de dois ângulos:

- $\theta \in [0, 2\pi]$ (longitude)
- $\phi \in [0, \pi]$ (latitude)

A conversão para coordenadas cartesianas é dada por:

\[
x = \sin(\phi)\cos(\theta)
\]

\[
y = \cos(\phi)
\]

\[
z = \sin(\phi)\sin(\theta)
\]

Como a esfera é inicialmente construída com raio unitário, todos os vértices encontram-se sobre a superfície da esfera de raio $1$ centrada na origem.

*** Discretização da Superfície

A superfície contínua é discretizada em duas direções:

- =stacks=: divisões horizontais (latitudes);
- =slices=: divisões verticais (longitudes).

Cada par $(stack, slice)$ gera um vértice da malha.

#+BEGIN_SRC text
      stack 0
   ----------------
   ----------------
   ----------------
      stack N
#+END_SRC

Quanto maior o número de subdivisões, mais suave será a esfera produzida, ao custo de uma quantidade maior de vértices e triângulos.

*** Normais

Para uma esfera unitária, o vetor normal em cada ponto coincide com a direção do próprio vértice.

Assim, para cada vértice $v$:

\[
n = \frac{v}{||v||}
\]

Como todos os vértices já pertencem à superfície de uma esfera de raio unitário, a normal pode ser obtida simplesmente normalizando a posição do vértice.

Isso produz iluminação suave (/smooth shading/) sobre toda a superfície.

*** Construção dos Triângulos

Após gerar os vértices, a superfície é triangulada.

Cada célula da grade formada por duas latitudes consecutivas e duas longitudes consecutivas é dividida em dois triângulos:

#+BEGIN_SRC text
A ----- B
| \     |
|   \   |
|     \ |
C ----- D
#+END_SRC

Triângulos gerados:

- $(A,C,B)$
- $(B,C,D)$

Esse procedimento é repetido para todas as células da malha.

*** Complexidade

Sejam:

- $S$ o número de =stacks=
- $L$ o número de =slices=

A quantidade de vértices gerada é:

\[
(S+1)(L+1)
\]

e a quantidade de triângulos é:

\[
2SL
\]

Portanto, o custo de geração da esfera é:

\[
O(SL)
\]

sendo executado apenas durante a criação da rocha.

*** Relação com o Sistema de Colisão

A utilização de uma esfera como geometria base permite empregar um modelo de colisão igualmente esférico.

Dessa forma, a colisão entre pássaro e rocha reduz-se ao teste da distância entre dois centros:

\[
||P - R|| < r_P + r_R
\]

onde:

- $P$ é o centro do pássaro;
- $R$ é o centro da rocha;
- $r_P$ é o raio de colisão do pássaro;
- $r_R$ é o raio de colisão da rocha.

Esse teste possui custo constante $O(1)$ e evita a necessidade de verificar interseções contra todos os triângulos da malha.

1.      Definir um arquivo /header/ para conter os objetos a serem desenhados; talvez torne-se interessante implementar diversos /headers/ no futuro, armazenando objetos conforme sua natureza na cena;
    1.  ver =./src/Objects.h/=
    2.  =Objects.h= implementa: 
        -   =VertexData=: estrutura {posição, vetor normal, coordenadas de  textura } para um vértice;
        -   =Mesh=: vetor de vértices, índices de vértices, modo de renderização (em triângulos via =GL_TRIANGLES=), vetores 3-dimensionais para desenhar uma caixa ao redor do objeto (bbox_{min, max}), identificadores para VAO, VBO, EBO para desenhar o objeto; 
        -   =SceneObject=: classe-base de alto-nível para representar um objeto na cena; contém o nome do objeto, transformações básicas (posição, rotação, escala), cor (ignorado se objeto tem textura), /flag/ booleana =visible= para indicar se o objeto é atualmente visível (renderizado) ou não;
            -   permite que objetos sejam acessados por seu nome;
        -   =Floor=: classe que herda de =SceneObject= para representar o /chão/ da cena. É inicializada com /length/ e /width/ para definir o tamanho do chão em relação aos eixos X, Z (?) respectivamente;
        -   O arquivo, atualmente, também implementa classes para *esfera* e *coelho* vistos em aula;
        -   =ObjectManager= permite gerenciar objetos na cena; adicionar e remover objetos, desenhar os objetos e obtê-los por nome.

2.      Implementar as classes e seus métodos em =./src/Objects.cpp/=
3.      Criar uma /factory/ para criação facilitada de objetos. A função da /factory/ é que poderemos fazer algo como =factory.createBirdObject(...params)= para criar um objeto do tipo =Bird=, e também =factory.createFloorObject(...params)= para criar um objeto do tipo chão (=Floor=). Uma /factory/ é uma abstração sobre o processo de criar novos objetos.
   1. Criar o /header/ para a fábrica em =ObjectFactory.h=. O arquivo /header/ já contém as implementações necessárias para a fábrica. Atualmente, a fábrica só tem três métodos: =createFloor()=, =createCheckeredFloor()= (outro tipo de chão, para testes) e =createSphere()=, para criar esferas.
4.       Em =main.cpp=, precisamos

incluir os headers que fizemos, fazendo
#+BEGIN_SRC cpp
#include "Objects.h"
#include "ObjectFactory.h"
#+END_SRC

inicializar um gerenciador de objetos
#+BEGIN_SRC cpp
ObjectManager& g_ObjectManager = ObjectManager::getInstance();
#+END_SRC

agora podemos criar os objetos /usando a fábrica/ ou /manualmente/:
#+BEGIN_SRC cpp
    // criar objetos usando a fábrica de objetos
    Floor* mainFloor = ObjectFactory::createCheckeredFloor(10.0f, 10.0f, -1.1f);
    g_ObjectManager.addObject(mainFloor);
    
    Sphere* redSphere = ObjectFactory::createSphere(0.5f, glm::vec3(-1.5f, 0.0f, 0.0f));
    g_ObjectManager.addObject(redSphere);
    
    Sphere* blueSphere = ObjectFactory::createSphere(0.5f, glm::vec3(1.5f, 0.0f, 0.0f));
    blueSphere->color = glm::vec4(0.2f, 0.2f, 0.8f, 1.0f);
    g_ObjectManager.addObject(blueSphere);

    // criar objetos manualmente
    Floor* customFloor = new Floor("custom_floor", 5.0f, 5.0f, -0.5f);
    customFloor->loadTexture("textures/wood.jpg");
    customFloor->position = glm::vec3(3.0f, 0.0f, 2.0f);
    g_ObjectManager.addObject(customFloor);
#+END_SRC

dentro do /loop/ de renderização, usamos o método para desenhar objetos do gerenciador de objetos:
#+BEGIN_SRC cpp
g_ObjectManager.drawAllObjects(g_model_uniform);
#+END_SRC

após o término do /loop/ de renderização, isto é, quando fecharmos a janela do jogo, é de boa prática limpar a memória do gerenciador, fazendo:
#+BEGIN_SRC cpp
g_ObjectManager.clear();
#+END_SRC
