#+title: Dividindo =main.cpp= em vários módulos

Atualmente, o arquivo principal do projeto (=main.cpp=) contém +1600 linhas. Apesar de muito bem estruturado, existem muitas responsabilidades distintas sendo mantidas em um único arquivo. Para fins práticos, a fim de nos encontrarmos com maior facilidade, propomos uma nova organização de arquivos. Atualmente, as responsabilidades de =main.cpp= são: /inicialização da janela, carregamento de shaders/objetos/texturas, callbacks de entrada (teclado, mouse), renderização de texto, lógica do jogo (controle do objeto principal) e loop principal. Isso dificulta a manutenção e a evolução do simulador./ A tendência é que muitas outras funcionalidades sejam construídas em cima destas e as consequências, se não bem administradas, podem ser catastróficas.

* Delimitação do Problema
O /problema/ atual é: /tudo/ em um único arquivo.
Proposta de solução (metas):
1. =main.cpp= deve conter apenas: criação da janela, inicialização dos subsistemas, loop principal (renderização + update), e desligamento.
2. Mover /callbacks/ de entrada para um módulo =InputHandler=.
3. Mover lógica da câmera para =Camera=.
4. Mover renderização de UI (texto, debug) para =UIRenderer=.
5. Mover carregamento de recursos (shaders, modelos, texturas) para =ResourceManager=.
6. Mover lógica do jogo (ave, pontuação, anéis) para =Game=.

#+begin_src cpp
// Camera.cpp
#include "Camera.hpp"
#include "matrices.h"

Camera::Camera() : theta(0), phi(0), distance(3.5f), lookAt(0,0,0) {}

void Camera::updateFromMouse(float dx, float dy, bool leftButton) {
    if (!leftButton) return;
    theta -= 0.01f * dx;
    phi   += 0.01f * dy;
    phi = std::clamp(phi, -3.14159f/2.0f, 3.14159f/2.0f);
}

void Camera::zoom(float delta) {
    distance -= 0.1f * delta;
    distance = std::max(distance, 0.01f);
}

glm::mat4 Camera::getViewMatrix() const {
    float x = distance * cos(phi) * sin(theta);
    float y = distance * sin(phi);
    float z = distance * cos(phi) * cos(theta);
    glm::vec4 pos(x, y, z, 1.0f);
    glm::vec4 look(lookAt, 1.0f);
    glm::vec4 up(0,1,0,0);
    return Matrix_Camera_View(pos, look - pos, up);
}
